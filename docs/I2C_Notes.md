# I²C (Inter-Integrated Circuit) Notes

## 1 Symbol Meanings

- `=>` : signifies **"This is" / assignment**
- `->` : signifies a **transition**
- `//` : comment
- `[]` : data sent by **master**
- `()` : data sent by **slave**

## 2️ SDA and SCL Lines

- **SDA** => Serial **DATA** Line
- **SCL** => Serial **CLOCK** Line

### Key Properties

- Both lines are **bidirectional**
- Both require **open-drain (open-collector) configuration**
- External **pull-up resistors** keep lines HIGH by default
- Devices pull lines LOW when transmitting

### Default Bus State (Idle)

    SDA => HIGH
    SCL => HIGH

## 3 SDA Transition Rules

SDA is only allowed to change state while **SCL is LOW**,
**except** during:

1. **START Condition**
2. **STOP Condition**

## Bus Conditions

### START Condition

**Definition:**
SDA transitions from HIGH -> LOW while SCL is HIGH.

**Generated by:** Master

    SDA => HIGH -> LOW
    SCL => HIGH

### STOP Condition

**Definition:**
SDA transitions from LOW -> HIGH while SCL is HIGH.

**Generated by:** Master

    SDA => LOW -> HIGH
    SCL => HIGH

This releases the bus and ends transmission.

### Repeated START

- Same electrical behavior as a normal START
- Occurs **without a preceding STOP**
- Keeps control of the bus

### Used For

- Changing data direction (Write -> Read)
- Retrying transmission
- Synchronizing devices
- Accessing serial memory

## Addressing

### Address Byte Format

    [7-bit Slave Address][R/W Bit]

- 7-bit address => Fixed by manufacturer
- R/W bit:
- `0` => Write
- `1` => Read

### Process

1. Master sends START
2. Master sends Address + R/W
3. Addressed slave responds with ACK
4. All other slaves ignore transaction and respond with NACK

## ACK and NACK

After **every 8-bit byte**, the receiver must respond.

### ACK (Acknowledge)

- Indicates successful reception
- Allows transmission to continue

        SDA => LOW
        SCL => HIGH

### NACK (Not Acknowledge)

- Indicates error or end of reading
- Master sends NACK after last byte when reading

        SDA => HIGH
        SCL => HIGH

### After a NACK, Master may

- Send STOP
- Send Repeated START
- Retry transmission

## Data Transmission

### Data Bits

- Transmitted in **8-bit bytes**
- **MSB first**
- Each bit synchronized with SCL

### Timing Rules

| SCL State | What Happens                  |
|-----------|------------------------------ |
| LOW       | Transmitter sets next bit     |
| HIGH      | Receiver reads bit            |

- SDA must remain stable while SCL is HIGH
- Receiver samples data when SCL is HIGH
- Transmitter changes data when SCL is LOW

## Command Byte

When accessing a specific register:

1. Master addresses slave
2. Master sends register address

Although technically data, this byte is often called a:

> **Command Byte**

It selects the internal register for read/write.

## Writing to a Device

Each line below represents 8 clock pulses.

    [START]

    [7-bit Address][R/W = 0]
    -> Wait for ACK

    [Command/Register Byte]
    -> Wait for ACK

    [Data Byte 1]
    -> Wait for ACK

    [Data Byte 2]
    -> Wait for ACK

    ...

    [Nth Data Byte]
    -> Wait for ACK

    [STOP]

(!) Some devices may require a Repeated START instead of STOP.

## Reading from a Device

(!) Many devices require a register write first before reading.

## Basic Read Sequence

    [START]

    [7-bit Address][R/W = 1]
    -> Wait for ACK

    (Data Byte 1) <- Slave drives SDA
    [ACK]

    (Data Byte 2)
    [ACK]

    ...

    (Nth Data Byte)
    [NACK]   <- Master signals done

    [STOP]

## Important Control Rule

- **SCL is ALWAYS controlled by the Master**
- SDA control switches between Master and Slave
- Master acts as traffic controller

## Quick Summary

- Bus idle => SDA = HIGH, SCL = HIGH
- START => SDA falls while SCL HIGH
- STOP => SDA rises while SCL HIGH
- Data changes when SCL LOW
- Data read when SCL HIGH
- Every byte requires ACK
- Master controls clock at all times
